## 3. Campo 结构展开

`build_campo_mother_field` 先从
`outputs/q2/07_最终方案摘要.json` 读取问题二 Campo 参数，确定性重建
1471 面完整结构，再把
`outputs/q2/03_最终镜位坐标.csv` 中的 1469 面正式镜场映射回完整结构。

每面镜子保存：

- `ring_indices`：1 起始圆环编号；
- `zone_indices`：Campo 区域 1 或 2；
- `zone_row_indices`：区域内 0 起始行号；
- `normalized_rows`：区域内归一化行号；
- `azimuth_angles`：相对塔正北方向的方位角；
- `azimuth_features`：同环中心化的 $\cos\theta$；
- `nominal_ring_counts`：所在环名义镜数；
- `actual_ring_counts`：裁剪和修剪后实际镜数；
- `original_indices`：完整 Campo 结构中的原编号。

启动时断言：

```text
正式镜数 = 1469
有效环数 = 28
区域环数 = (11, 17)
区域镜数 = (624, 845)
```

如果问题二参数、坐标或 Campo 生成规则变化，程序停止，而不是静默使用错误
标签。

## 4. 连续设计状态

设计状态由 `ContinuousDesign` 保存：

```python
ContinuousDesign(
    size_zone1_slope=...,
    size_zone2_slope=...,
    size_zone2_offset=...,
    height_offset=...,
    height_zone1_slope=...,
    height_zone2_slope=...,
    height_zone2_offset=...,
    size_azimuth=...,
    height_azimuth=...,
    area_ratio=...,
)
```

`expand_continuous_design` 先计算逐镜尺寸形状 $g_i$，再解析求解
`area_normalizer`，最后得到逐镜 `widths`、`heights`、
`installation_heights` 和 `areas`。固定面积阶段不会因为浮点搜索产生面积
漂移。

## 5. 几何预筛选

每个候选在光学评价前检查：

- 数组长度、有限性和正值；
- $2\le h_i\le w_i\le8$；
- $2\le H_i\le6$；
- $H_i\ge h_i/2$；
- 场地半径不超过 350 m；
- 塔周距离不小于 100 m；
- $d_{ij}>\max(w_i,w_j)+5+0.01$。

由于镜面宽度上限为 8 m，只需对中心距不超过 13.01 m 的镜对执行精确
异构宽度检查。

本轮实验发现第一环宽度已贴近问题二镜间距上限。固定总面积且强制尺寸向外
减小时，内侧镜面会被面积归一化系数放大，容易首先违反该约束。这是最终
执行单调放松检验的直接原因。

## 6. 搜索流程

默认搜索顺序为：

1. 1469 面统一规格回归；
2. 纯径向单调模型的安装高度搜索；
3. 固定总面积的尺寸形状搜索；
4. 总面积比例逐级压缩；
5. 自动方位诊断；
6. 非零方位尺寸和高度参数的整场消融；
7. 尺寸与高度趋势的小范围单调放松；
8. 少量模型正式精度比较；
9. 可选外边界对称删镜；
10. 加密精度和邻镜半径敏感性验证。

粗精度差分用于候选排序，候选接受以中精度完整评价为准。若当前中精度基线
因离散偏差暂时低于 42 MW，搜索先接受能提高功率的高度候选，进入可行域后
再按功率约束下的 $q$ 比较。

## 7. 方位项的两层判断

`diagnose_campo_structure` 先对统一规格单镜单位面积输出做两个最小二乘
诊断：

```text
区域截距 + 区域内行号
区域截距 + 区域内行号 + 同环中心化方位
```

本轮 RMSE 相对下降 42.333%，因此自动模式会启动方位参数搜索。但是尺寸
方位项和高度方位项均未在完整镜场中获得可接受的中精度提升，最终仍取零。

这说明单镜解释能力只能用于提出候选，不能替代包含镜间耦合的整场目标。

## 8. 运行命令

所有 Python 命令使用 Conda `agent` 环境。

快速端到端烟雾测试：

```bash
conda run -n agent env PYTHONPATH=src \
python src/solve_q3_continuous.py \
  --smoke \
  --max-cycles 1 \
  --prune-rounds 0 \
  --output /tmp/q3-continuous-smoke
```

独立正式搜索：

```bash
conda run -n agent env PYTHONPATH=src \
python src/solve_q3_continuous.py
```

增加加密验证：

```bash
conda run -n agent env PYTHONPATH=src \
python src/solve_q3_continuous.py --run-validation
```

可选参数：

| 参数 | 含义 |
| --- | --- |
| `--full-campo-prefix` | 改用修剪前 1471 面完整 Campo 前缀 |
| `--azimuth-mode auto` | 由单镜诊断决定是否搜索方位项 |
| `--azimuth-mode on` | 强制执行方位项消融 |
| `--azimuth-mode off` | 跳过方位项 |
| `--skip-relaxed` | 跳过单调放松检验 |
| `--max-cycles` | 每个步长的最大坐标扫描轮数 |
| `--prune-rounds` | 外边界对称删镜最大轮数 |

运行测试：

```bash
conda run -n agent env PYTHONPATH=src \
python -m unittest discover -s tests -v
```

本轮独立正式搜索使用：


```text

总面积：61300.700 m²
```

加密验证为：

```text

```

同一连续参数作用于 1471 面完整前缀时，正式
$q=0.687844\ \mathrm{kW/m^2}$，因此外边界选择保留 1469 面。

## 10. 结果边界

连续方案相对问题二统一规格的正式 $q$ 提高约 1.008%，但比原六组方案低
约 0.572%。正式结论必须同时保留这两项比较：

- 连续 Campo 参数化证明了不依赖人工六组边界也能获得稳定改善；
- 当前预算下，连续方案尚未在数值上超过原六组方案。

不要把烟雾测试结果写入论文，也不要把单镜方位拟合改善表述为整场方位优化
收益。
